# PC Quanti MVP - UI/UX Design Audit
**Date:** 2026-02-14 17:44:29  
**Scope:** Design-only audit (visual design + interaction design)  
**Target:** PC Quanti MVP website (Next.js + Tailwind CSS)  
**Philosophy:** Jobs/Ive design principles - crisp, uncluttered, services-centric

---

## Executive Summary

**Current State:** Boilerplate Next.js starter with default template content  
**Target State:** Professional B2B consultancy website optimized for nuclear/industrial project management leads  
**Primary Goal:** Transform generic starter into a crisp, trust-building service showcase

**Key Findings:**
1. ✅ **Strong foundation:** Next.js + Tailwind CSS provides excellent performance baseline
2. ⚠️ **Brand mismatch:** Current design uses generic Next.js branding vs. PC Quanti's bright blue (#2C02D9) identity
3. ⚠️ **Content void:** Placeholder content doesn't communicate professional services value
4. ⚠️ **Navigation missing:** No way to discover 5 core services or reach contact form
5. ✅ **Mobile-first structure:** Responsive flex layout is solid starting point

---

## Audit Findings by Component

### 1. Brand Identity & Color System

**Current:**
- Next.js logo and branding
- Zinc/neutral color palette (bg-zinc-50, text-zinc-600)
- Generic dark mode support
- No PC Quanti brand presence

**Issues:**
- ❌ Missing brand color #2C02D9 (bright blue from logo)
- ❌ Logo file exists at `/public/logo.png` but not used
- ❌ Color system doesn't support professional consultancy positioning

**Recommendations:**
- **Phase 1a:** Replace Next.js logo with PC Quanti logo (`/public/logo.png`)
- **Phase 1b:** Add brand color to Tailwind config as `pc-blue` or `pc-cta`
- **Phase 1c:** Update color palette to professional B2B scheme:
  - Primary CTA: #2C02D9 (bright blue)
  - Neutral text: Refined grays (900/700/500 for hierarchy)
  - Backgrounds: Clean whites/off-whites (avoid pure black in dark mode for softer feel)

**Design Philosophy Note:** Jobs/Ive approach = bold accent color used sparingly on critical CTAs, surrounded by generous whitespace and neutral tones. Perfect fit for PC Quanti's "crisp, not cluttered" requirement.

---

### 2. Typography System

**Current:**
- `font-sans` (default system fonts)
- Good hierarchy: h1 @ 3xl, body @ lg
- Tracking and leading values are reasonable

**Issues:**
- ❌ No custom typeface (plan specifies Inter or Manrope)
- ⚠️ Headings could use tighter tracking for professional feel
- ✅ Font sizes and weights are sensible

**Recommendations:**
- **Phase 1d:** Import Google Fonts (Inter recommended for B2B clarity)
- **Phase 1e:** Refine typography scale:
  - H1: 4xl-5xl for landing sections (hero)
  - H2: 3xl for section headers
  - H3: 2xl for service cards
  - Body: base-lg for readability
  - Tight tracking (-0.02em) on headings for polish

**Design Philosophy Note:** Ive's typography = clarity first, generous sizing for important content, tight tracking for premium feel. Inter's neutrality won't compete with service messaging.

---

### 3. Layout & Spacing

**Current:**
- Centered max-w-3xl container
- py-32 px-16 spacing (generous)
- Vertical flex layout with gap-6

**Issues:**
- ⚠️ Max-width too narrow for multi-column service showcase
- ⚠️ Excessive top/bottom padding wastes hero space
- ✅ Gap values are clean

**Recommendations:**
- **Phase 2a:** Expand max-width to 7xl for full-width hero, 5xl for content sections
- **Phase 2b:** Reduce hero padding to py-16 md:py-24 (mobile-first)
- **Phase 2c:** Implement section-based spacing system:
  - Hero: py-16 md:py-24
  - Content sections: py-12 md:py-20
  - Footer: py-8 md:py-12

**Design Philosophy Note:** Jobs presentation rule = don't waste space on padding, use it for content that drives action. Hero should capture attention immediately, not force scrolling.

---

### 4. Component Architecture

**Current:**
- Inline JSX in page.tsx
- No reusable components
- Hardcoded Vercel CTAs

**Issues:**
- ❌ No component library (Button, Card, Container required per plan)
- ❌ No navigation system
- ❌ CTAs point to external Vercel sites vs. PC Quanti goals

**Recommendations:**
- **Phase 2d:** Extract reusable components to `src/components/`:
  - `Button.tsx` (primary/secondary variants, using brand color)
  - `Card.tsx` (for service cards with hover states)
  - `Container.tsx` (max-width wrapper with consistent padding)
  - `Nav.tsx` (header navigation)
  - `Footer.tsx` (contact info, legal links)
- **Phase 2e:** Build service card component:
  - Clean white/gray background
  - Icon or small graphic (optional)
  - Service name (h3)
  - 2-3 sentence description
  - Subtle hover effect (shadow or border color change)
  - Link to detail page

**Design Philosophy Note:** Ive components = minimal surface complexity, maximum functional clarity. Each component should have single clear purpose.

### 7. Mobile Experience

**Current:**
- Responsive flex layout
- sm: breakpoint switches alignment
- Mobile-first spacing (flex-col → sm:flex-row)

**Issues:**
- ⚠️ Padding too large on mobile (px-16 wastes precious screen space)
- ✅ Flex direction switching works well

**Recommendations:**
- **Phase 3e:** Refine mobile spacing:
  - Container: px-4 md:px-8 lg:px-12
  - Typography: text-2xl md:text-4xl for h1
  - Buttons: Full-width on mobile, auto-width on desktop
- **Phase 3f:** Touch targets:
  - Minimum 44x44px for all interactive elements
  - Generous spacing between nav links on mobile menu
  - Sticky header height: 64px mobile, 80px desktop

**Design Philosophy Note:** Ive mobile rule = design for thumb, not mouse. Every interaction should feel natural on a phone.

---

### 8. Accessibility

**Current:**
- Semantic HTML (main, a, Image)
- Dark mode support
- Alt text on images

**Issues:**
- ⚠️ Dark mode color contrast may fail WCAG AA (pure black backgrounds)
- ⚠️ No skip-to-content link
- ⚠️ No focus indicators on links

**Recommendations:**
- **Phase 3g:** WCAG AA compliance:
  - Add focus-visible rings to all interactive elements
  - Skip-to-content link (visually hidden until focused)
  - Contrast check: text-zinc-600 on zinc-50 passes, but verify all color combinations
  - Alt text for logo: "PC Quanti logo"
- **Phase 3h:** Dark mode refinement:
  - Use zinc-900 instead of pure black
  - Ensure 4.5:1 contrast ratio for body text
  - Test with browser accessibility tools

**Design Philosophy Note:** Ive accessibility = inclusive by default, not bolted on. Good design serves all users.

---

### 9. Performance & Technical

**Current:**
- Next.js App Router
- Image optimization via next/image
- Priority loading on logo

**Issues:**
- ✅ Excellent foundation
- ⚠️ No font optimization strategy yet
- ⚠️ No images beyond logos

**Recommendations:**
- **Phase 3i:** Performance optimization:
  - Use next/font for Google Fonts (eliminates layout shift)
  - Optimize logo.png (current size unknown, aim for <20KB)
  - Add service icons as SVGs (inline for critical path)
  - Lazy load footer/secondary content
- **Phase 3j:** Lighthouse targets:
  - Desktop: 95+ (easily achievable with static site)
  - Mobile: 85+ (font loading is main bottleneck)
  - LCP: <2.5s (hero should render immediately)

## Design Philosophy Summary

**Jobs/Ive Principles Applied:**

1. **Clarity over cleverness:** No animations or interactions that don't serve user goal (reaching contact form)
2. **Whitespace as design element:** Generous spacing makes services stand out, not compete
3. **Color discipline:** Bright blue (#2C02D9) reserved for primary CTAs only—everything else neutral
4. **Mobile-first thinking:** Design for thumb-tap interaction, scale up to desktop
5. **Content is king:** Every pixel serves the conversion goal (visitor → lead)
6. **Performance = feature:** Fast loading builds trust in professional services context
7. **Accessible by default:** Focus indicators, semantic HTML, keyboard navigation—not afterthoughts

**PC Quanti Specific:**

- **"Crisp, not cluttered":** Minimal chrome, maximum signal. Services should be discoverable immediately.
- **"Services-centric":** Home page is a launchpad to 5 service pages, not a novel.
- **Trust without stuffiness:** Professional but approachable. Phil's lean model is a feature, not a bug.
- **B2B conversion focus:** Every design decision optimized for "industrial project manager finds PC Quanti via Google → contacts within 2 minutes."
