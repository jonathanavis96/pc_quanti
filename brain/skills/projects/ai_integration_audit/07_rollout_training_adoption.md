# Rollout, Training, Adoption (Make It Stick)

## Why This Exists

Even good automations fail if they aren’t adopted. This skill provides a rollout + enablement framework.

## When to Use It

Use once initial workflows are live (shadow/approval mode) and you’re preparing handover.

## Details

### Rollout Phases

- Pilot with 1–3 power users
- Collect feedback + update prompts/SOPs
- Expand to team
- Raise autonomy only when metrics support it

### Training Modules (role-based)

- End users: how to request, review, and approve outputs
- Managers: how to monitor metrics and exceptions
- Admin/ops: how to update KB, prompts, and workflows

### Adoption Metrics

Track:

- usage (runs/day)
- time saved (self-reported + measured)
- quality (rewrite rate, escalation rate)
- business outcome metrics (CSAT, lead response time)

### SOP Updates

Every deployed workflow must have:

- When to use it
- What good looks like
- How to handle failures
- escalation path

## Deliverables

Done when:

- Training delivered + recorded
- SOPs updated
- Ownership transferred with monitoring in place
